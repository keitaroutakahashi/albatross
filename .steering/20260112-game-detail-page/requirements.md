# 要求内容

## 概要

ゲーム詳細画面（/games/[id]）を作成し、試合のスコアボードと詳細情報を表示する機能を実装します。

## 背景

現在、試合一覧画面（/games）は実装されていますが、各試合の詳細を確認できる画面がありません。ユーザーが試合一覧から特定の試合をクリックした際に、イニングごとのスコアや試合の詳細情報を確認できるようにする必要があります。

## 実装対象の機能

### 1. 試合詳細ページ（/games/[id]）

- 試合の基本情報（日付、リーグ名、グラウンド）を表示
- イニングごとのスコアボードを表示
- 両チームの最終スコアを表示
- 試合結果（勝敗）を視覚的に表示

### 2. 試合一覧からの遷移

- 試合一覧の各アイテムをクリックすると詳細ページへ遷移
- 詳細ページから一覧へ戻る導線を提供

## 受け入れ条件

### 試合詳細ページ

- [ ] URL `/games/[id]` でアクセスできる
- [ ] 試合の基本情報（日付、曜日、年、リーグ名）が表示される
- [ ] グラウンド名が表示される
- [ ] スコアボード（イニングごとの得点テーブル）が表示される
- [ ] 両チームの合計得点が表示される
- [ ] 勝敗が視覚的に分かる（勝ち: 緑、負け: 赤、引き分け: グレー）
- [ ] 試合一覧へ戻るリンクがある

### 試合一覧からの遷移

- [ ] 試合一覧の各アイテムをクリックすると詳細ページに遷移する
- [ ] リンクが正しく機能する

## 成功指標

- 試合詳細ページが 1 秒以内にロードされる
- モバイル・デスクトップ両方でスコアボードが見やすく表示される

## スコープ外

以下はこのフェーズでは実装しません:

- 打撃成績・投手成績の表示（別タスクで実装）
- 成績編集機能
- データベース連携（ダミーデータを使用）
- 認証・権限管理
